# DEC-0025: The system is named Groundwork

## Context

The system needed a name to anchor the repository, documentation, and
glossary.

## Decision

The system is named **Groundwork**.

## Rationale

Evokes the core value proposition: grounding implementation in business
intent, with documentation as the foundation everything is built on.

## Alternatives Considered

- **Provenance**: emphasizes the transcript→decision→contract lineage.
- **Keystone**: the gate/arch metaphor.

## Implications

"Groundwork" is the glossary's first entry ([CONTEXT.md](../../CONTEXT.md)).
