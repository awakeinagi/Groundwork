# IDEA-0057: Decide this repository's operating mode once the Application exists

## The Idea

When the Groundwork Application exists, this repository (the Groundwork project's own corpus) must choose whether to adopt the Application, stay skill-mode-only as the permanent dogfood of the standalone path, or sequence between the two. The trade-off to weigh then is that adopting the app gives the best application dogfood while retaining skill-only use preserves the only pure standalone proving ground.

## Spark Context

Captured at SES-0082 T42: during round-two grilling close-out, the stakeholder was asked whether this repository adopts the Application when it exists or stays skill-mode-only as the standalone dogfood, and deliberately deferred the choice until the Application is real rather than deciding it now.

## Disposition

Captured, pending the Application's existence. Revisit this choice once BG-0002's Application track produces a usable instance; until then, this repository continues operating skill-mode-only.
